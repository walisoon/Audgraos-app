# Configuração do Supabase

Este aplicativo agora usa o Supabase como banco de dados principal para armazenar informações de usuários.

## Passos para Configurar

### 1. Criar Projeto Supabase

1. Acesse [https://supabase.com](https://supabase.com)
2. Crie uma conta ou faça login
3. Clique em "New Project"
4. Dê um nome ao seu projeto (ex: "audgraos-app")
5. Escolha uma senha de banco de dados
6. Selecione a região mais próxima
7. Clique em "Create new project"

### 2. Obter Credenciais

Após criar o projeto, você precisará de duas informações:

1. **Project URL**: Encontre em Settings > API
2. **anon/public key**: Também em Settings > API

### 3. Configurar Tabela de Usuários

No painel do Supabase:

1. Vá para "Table Editor"
2. Clique em "Create a new table"
3. Configure a tabela:

```sql
CREATE TABLE users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  senha TEXT NOT NULL,
  nome TEXT NOT NULL,
  tipo TEXT NOT NULL DEFAULT 'auditor',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Criar índice para busca por email
CREATE INDEX idx_users_email ON users(email);
```

Ou use a interface visual:
- **Table name**: `users`
- **Columns**:
  - `email` (text, unique, required)
  - `senha` (text, required)
  - `nome` (text, required)
  - `tipo` (text, default: 'auditor')
  - `created_at` (timestamptz, default: now())
  - `updated_at` (timestamptz, default: now())

### 4. Atualizar Credenciais no App

Abra o arquivo `lib/services/supabase_service.dart` e substitua:

```dart
static const String _supabaseUrl = 'SUA_URL_SUPABASE_AQUI';
static const String _supabaseAnonKey = 'SUA_CHAVE_ANONIMA_SUPABASE_AQUI';
```

Com suas credenciais reais:

```dart
static const String _supabaseUrl = 'https://seu-projeto.supabase.co';
static const String _supabaseAnonKey = 'sua-chave-anonima-aqui';
```

### 5. Configurar Políticas de Segurança (RLS)

No painel do Supabase:

1. Vá para "Authentication" > "Policies"
2. Para a tabela `users`, crie as seguintes políticas:

**Policy 1: Permitir leitura para todos usuários autenticados**
```sql
CREATE POLICY "Users can view all users" ON users
  FOR SELECT USING (auth.role() = 'authenticated');
```

**Policy 2: Permitir inserção para todos usuários autenticados**
```sql
CREATE POLICY "Users can insert users" ON users
  FOR INSERT WITH CHECK (auth.role() = 'authenticated');
```

**Policy 3: Permitir atualização para todos usuários autenticados**
```sql
CREATE POLICY "Users can update users" ON users
  FOR UPDATE USING (auth.role() = 'authenticated');
```

**Policy 4: Permitir exclusão para todos usuários autenticados**
```sql
CREATE POLICY "Users can delete users" ON users
  FOR DELETE USING (auth.role() = 'authenticated');
```

### 6. Testar a Conexão

Execute o aplicativo. Ele deve:

1. Conectar-se ao Supabase automaticamente
2. Criar usuários padrão (admin@audgraos.com, usuario@audgraos.com)
3. Permitir adicionar, editar e excluir usuários
4. Sincronizar dados em tempo real

## Benefícios da Migração

✅ **Dados persistentes**: Não perde dados ao reinstalar o app  
✅ **Sincronização multi-dispositivo**: Mesmos dados em vários dispositivos  
✅ **Backup automático**: Supabase mantém backups  
✅ **Escalabilidade**: Suporta múltiplos usuários simultâneos  
✅ **Segurança**: RLS (Row Level Security) para controle de acesso  

## Solução de Problemas

### Erro de Conexão
- Verifique se as credenciais estão corretas
- Confirme se o projeto Supabase está ativo
- Verifique a conexão com a internet

### Erro de Permissão
- Verifique se as políticas RLS estão configuradas
- Confirme se a autenticação está funcionando

### Dados Não Aparecem
- Verifique se a tabela `users` existe
- Confirme se os usuários padrão foram criados
- Verifique o console do aplicativo para erros
